# MimiClaw 编译烧录指南（SenseCAP Watcher）

## 环境要求

- ESP-IDF v5.5+（项目依赖 IDF 5.5 及以上版本）
- 目标芯片：ESP32-S3（16MB Flash / 8MB PSRAM）

---

## 第一步：激活 IDF 环境

```bash
conda activate idf55
source /opt/esp-idf/export.sh
newgrp uucp
```

---

## 第二步：设置编译目标（首次或 fullclean 后执行）

```bash
idf.py set-target esp32s3
```

---

## 第三步：配置 secrets（首次必做）

```bash
cp main/mimi_secrets.h.example main/mimi_secrets.h
```

编辑 `main/mimi_secrets.h`，填写 WiFi / Telegram / API Key 等信息：

```c
#define MIMI_SECRET_WIFI_SSID       "你的WiFi名"
#define MIMI_SECRET_WIFI_PASS       "你的WiFi密码"
#define MIMI_SECRET_TG_TOKEN        "123456:ABC-DEF..."
#define MIMI_SECRET_API_KEY         "sk-ant-api03-xxxxx"
#define MIMI_SECRET_MODEL_PROVIDER  "anthropic"   // 或 "openai"
#define MIMI_SECRET_SEARCH_KEY      ""            // 可选：Brave Search API Key
#define MIMI_SECRET_PROXY_HOST      ""            // 可选：HTTP 代理 host
#define MIMI_SECRET_PROXY_PORT      ""            // 可选：HTTP 代理 port
```

> **注意**：修改 `mimi_secrets.h` 后，必须执行 `idf.py fullclean` 再重新编译，否则更改不会生效。

---

## 第四步：编译

```bash
# 正常编译
idf.py build

# 修改过 mimi_secrets.h 后必须完全重编译
idf.py fullclean && idf.py build
```

编译成功后会生成 SPIFFS 镜像并打包到固件中（CMakeLists.txt 中有 `spiffs_create_partition_image`）。

---

## 第五步：确认串口设备

SenseCAP Watcher 只有一个 USB-C 口，接的是 CP2102 UART 桥接芯片。

```bash
# Linux
ls /dev/ttyUSB*      # 通常为 /dev/ttyUSB0
```

如果没有识别到，检查 CP2102 驱动是否已安装（Arch Linux 通常内核自带）。

---

## 第六步：烧录 + 监视串口

```bash
# 烧录并打开串口监视（替换 PORT 为实际端口）
idf.py -p /dev/ttyUSB0 flash monitor

# 只烧录不监视
idf.py -p /dev/ttyUSB0 flash

# 只打开串口监视（不烧录）
idf.py -p /dev/ttyUSB0 monitor
```

退出串口监视：按 `Ctrl + ]`

---

## 分区说明

本项目使用自定义分区表 `partitions.csv`（16MB Flash）：

| 分区      | 类型  | 偏移       | 大小   | 说明              |
|-----------|-------|------------|--------|-------------------|
| nvs       | data  | 0x9000     | 24KB   | 运行时配置（NVS） |
| otadata   | data  | 0xF000     | 8KB    | OTA 状态          |
| phy_init  | data  | 0x11000    | 4KB    | RF 校准数据       |
| ota_0     | app   | 0x20000    | 2MB    | 主应用固件        |
| ota_1     | app   | 0x220000   | 2MB    | OTA 备份固件      |
| spiffs    | data  | 0x420000   | ~12MB  | 文件系统（记忆、配置）|
| coredump  | data  | 0xFF0000   | 64KB   | Crash dump        |

## 运行时 CLI 配置（无需重新编译）

烧录完成后，通过串口 REPL 可随时修改配置（保存到 NVS，优先级高于编译时 secrets）：

```
mimi> wifi_set MySSID MyPassword     # 修改 WiFi
mimi> set_api_key sk-ant-api03-...   # 修改 API Key
mimi> set_tg_token 123456:ABC...     # 修改 Telegram Token
mimi> set_model_provider openai      # 切换 AI 提供商
mimi> set_proxy 127.0.0.1 7979       # 设置 HTTP 代理
mimi> config_show                    # 查看当前配置
mimi> config_reset                   # 恢复出厂配置（清除 NVS）
mimi> restart                        # 重启
```

## 常见问题

**Q: 烧录失败 / 连接超时**
- 检查串口号是否正确（`ls /dev/ttyUSB*`）
- SenseCAP Watcher 进入下载模式：按住 boot 键再按 reset，松开 reset 后松开 boot

**Q: 修改 WiFi / API Key 后没有生效**
- 修改 `mimi_secrets.h` 必须 `fullclean` 重编译
- 若已运行，优先用 `config_show` 检查 NVS 里是否有旧值覆盖了 secrets

**Q: SPIFFS 数据丢失**
- `idf.py flash` 只烧录 app 分区，不会擦除 SPIFFS
- `idf.py erase-flash` 会清除整块 Flash，包括 NVS 和 SPIFFS
